## Original Symptom (Before Fix)

When opening PPTX files generated by svg2ooxml in PowerPoint:
```
PowerPoint found a problem with content in presentation.pptx.
PowerPoint can attempt to repair the presentation.
```

PowerPoint could repair the file, but this indicated non-compliance with OOXML spec.

## Root Cause Analysis

Compared original vs. PowerPoint-repaired PPTX files:

### 1. Missing Required OOXML Files

PowerPoint expects these files which we don't generate:

- `/ppt/presProps.xml` - Presentation properties
- `/ppt/viewProps.xml` - View properties
- `/ppt/tableStyles.xml` - Table styles

**Impact**: HIGH - These are part of the ECMA-376 specification for PresentationML.

### 2. Font Embedding Issues

**Current behavior**:
- Fonts embedded in `/ppt/fonts/*.ttf`
- Content type: `application/x-font-ttf` (non-standard)
- Font declarations in `[Content_Types].xml`

**PowerPoint's fix**:
- Removed `/ppt/fonts/` directory entirely
- Removed font content type declarations
- Reverted to default fonts

**Impact**: MEDIUM - Fonts don't work, but content preserved.

**Correct approach**: According to ECMA-376, embedded fonts should use:
- Content type: `application/x-fontdata` (or reference external fonts)
- Proper font relationships in `.rels` files
- Font parts under `/ppt/fonts/` or `/word/fonts/`

### 3. Hyperlink Format Issues - ✅ FIXED

**Problem**: Invalid `ppaction://` URLs with unsupported parameters.

**Original behavior**:
```xml
<a:hlinkClick history="1" action="ppaction://hlinkshowjump?bookmark=testrect1"/>
```

**PowerPoint's fix**:
```xml
<a:hlinkClick r:id=""/>
```

**Root Cause**: Per ECMA-376, `ppaction://hlinkshowjump` only supports the `jump` parameter with values: nextslide, previousslide, firstslide, lastslide, endshow. The `bookmark` and `show` parameters are NOT part of the specification.

**Solution** (`src/svg2ooxml/drawingml/navigation.py`):
- Removed invalid `ppaction://hlinkshowjump?bookmark=...` URL generation
- Removed invalid `ppaction://hlinkshowjump?show=...` URL generation
- Only ACTION navigation (next/previous slide) generates `ppaction://` URLs
- BOOKMARK and CUSTOM_SHOW navigation don't have direct PowerPoint equivalents

**Tests**:
- `tests/unit/drawingml/test_navigation_ppaction.py` (6 tests)
- `tests/unit/io/test_pptx_required_parts.py::test_hyperlink_format_does_not_use_invalid_ppaction`

**Impact**: FIXED - Hyperlinks now use valid PowerPoint formats or are omitted if no equivalent exists.

### 4. Missing DocProps Declarations

**Current**: `[Content_Types].xml` doesn't declare `/docProps/` overrides

**PowerPoint added**:
```xml
<Override PartName="/docProps/core.xml" ContentType="application/vnd.openxmlformats-package.core-properties+xml"/>
<Override PartName="/docProps/app.xml" ContentType="application/vnd.openxmlformats-officedocument.extended-properties+xml"/>
```

**Impact**: LOW - Files exist but not properly declared.

## Recommended Fixes

### Priority 1: Add Missing Required Files

```python
# In PPTXPackageBuilder
def _add_required_parts(self):
    # Add presProps.xml
    self._add_presentation_properties()

    # Add viewProps.xml
    self._add_view_properties()

    # Add tableStyles.xml
    self._add_table_styles()
```

Minimal valid content for each:
